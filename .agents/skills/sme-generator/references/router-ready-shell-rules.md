# Router-Ready Shell Rules

SME Generator is Layer 2. It creates router-ready shells, not the router and not final knowledge packs.

The future SME Answer Router / Orchestrator should be able to read each shell and determine:

1. What this SME knows right now.
2. When this SME should be invoked.
3. What questions this SME can help answer now.
4. What questions this SME cannot answer yet.
5. What records this SME would ask for.
6. What documents would level this SME up.
7. What evidence would level this SME up.
8. What boundaries this SME must obey.
9. What contribution this SME should provide when invoked.

## Required Shell Sections

Every profile shell must include:

- SME Identity
- Required Disclaimer
- What This SME Knows Now
- When To Route Questions To This SME
- Questions This SME Can Help Answer Now
- Questions This SME Cannot Answer Yet
- Records This SME Would Ask For
- Documents That Would Level This SME Up
- Evidence That Would Level This SME Up
- Evidence Boundaries
- Router Contribution Format
- Open Questions

## SME Type Requirements

Shop Flow SME must help with route order, upstream/downstream context, operation ownership, work centers, and machine-to-operation mappings.

Inspection SME must help with inspection points, acceptance documents, evidence needs, before/after detection questions, and named inspection/test machines.

Machine / Capability SMEs must help with their named machine/capability, related operations, related work centers, likely records, level-up documents, evidence needs, and handoffs.

## Boundaries

- Do not build the router.
- Do not build knowledge packs.
- Do not perform RCCA.
- Do not perform root cause analysis.
- Do not recommend corrective action.
- Do not add deep technical knowledge.
- Do not web scrape.
- Do not create a vector database.
