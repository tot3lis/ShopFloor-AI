# Synthetic Injection Molding Shop Example

This example is fully synthetic. It does not represent a real shop, customer, product, router, part number, machine list, inspection plan, or production record.

The `outputs/` folder demonstrates what generated ShopFloor AI outputs can look like after the Layer 1-4 pipeline has run:

- `shop-reference.md`
- `sme-coverage.md`
- `sme-registry.md`
- `smes/*.md`
- `knowledge-package-registry.md`
- `sme-knowledge-map.md`
- `knowledge-packages/*`

Do not edit this example to onboard your own shop.

To onboard your own shop, drop messy shop files directly into Codex, then type `run shopfloor-ai`. You may also place files in `inputs/` if you prefer, but the folder is optional.
